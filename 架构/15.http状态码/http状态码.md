## http状态码
- 101  websock
- 200 成功 204 (没有响应体) 206(范围请求) 一般是断点下载
- 304 对比缓存 301 永久重定向 302 临时重定向 seo优化 307
- 400(前端参数传递错误) 401(没登录) 403(登陆了但是没权限) 404(找不到路径) 405(服务器不支持此方法)
- 500 502 503 服务端错误


## 缓存(服务端设置) 
- 强制缓存
  - 不会像服务器发送请求(当前访问的页面不走强制缓存,内部引用资源走强制缓存)
  - html不缓存也不支持缓存 (css/image/js)走缓存
    - 流程: 
      - 1. 先请求html 服务器返回html
      - 2. 有了html,就把里面的js/css/image请求 服务器返回
      - 3. 再次请求html 服务器返回html
      - 4. 再次根据html里的请求css/js/image 但是发现里面设置了cache-control(缓存头) expries(老版缓存头) 就让浏览器去缓存memory里查找了  一般都是设置缓存过期时间的

```
这两个是强制缓存,根据时间来控制,时间不到就不会发送请求
res.setHeader('cache-control','max-age=10') // 设置缓存头,10秒内不再重新请求css/image/js 
res.setHeader('expires',new Date(Date.now()+10*1000).toGMTString()) // 低版本浏览器的写法
memory cache disk cache就是从缓存里找的样式表 浏览器会根据使用次数文件类型自动去存储到缓存还是硬盘里 
```
