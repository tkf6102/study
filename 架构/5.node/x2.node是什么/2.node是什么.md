## node是什么,可以做什么
1. 可以让js运行在服务端
模块的特性,有一些api的包
实现文件操作
服务端的创建
2. 多线程(还是保留主线程的概念)
每次发送请求都会创建一个线程,而且可以处理多线程,但是不是同时处理,而是类似于事件轮询的样子,谁处理完了就切换到谁的位置,将其数据处理,然后再去看是否有完成的事件

切换时间片的概念 
  坏处 => 比较浪费性能
  好处 => 适合大量的计算,不阻塞.  适合cpu密集型(需要多个cpu帮忙计算) cpu密集: 压缩/计算/合并
线程池:
  创建线程一般是20个

3. 单线程 
    好处: 高并发,i/o密集
    坏处: 不合适cpu密集型.

4. node跨域解决
处理服务端数据
    (前端页面 =发请求=>node) ===>java代码
    因为前端代码是请求node页面,这里面是不存在跨域问题,而跨域在node中解决.
node解决数据处理
    也是在处理跨域的时候,代码返回到node里,处理数据.再给到页面. 就感觉node比较重型
