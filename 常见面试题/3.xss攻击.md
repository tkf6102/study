1)Xss跨站脚本攻击
就是尽一切努力把恶意代码注入到网页中
- 存储型(server端)
```
场景: 常见于网站有保存用户信息的功能,例如: 评论区,留言板,用户私信等
攻击步骤: 
    1. 存储于目标网站的数据库中
    2. 用户访问网站,数据库把代码返回到前端浏览器上
    3. 用户收到请求数据,立刻开始解析,混在其中的恶意代码也被解析.
    4. 恶意代码可以窃取用户数据发送到指定网站或者冒充用户发送请求
```

- 反射型(server端)
与存储型的代码区别在于,存储型代码是存储在数据库里,反射型代码是存储在URL中
```
场景: 通过url传递参数,如网站搜索,跳转等
    1. 黑客构造出特殊url(邮件发给用户个特殊链接),其中包含恶意代码
    2. 用户访问url,黑客指定的服务器返回给用户恶意代码
    3. 用户收到请求数据,立刻开始解析,混在其中的恶意代码也被解析.
    4. 恶意代码可以窃取用户数据发送到指定网站或者冒充用户发送请求
```
- DOM(浏览器端)
DOM型XSS攻击,属于前端取出和执行 属于JavaScript自身漏洞,而其他两种属于服务器端的漏洞
```
场景: 通过url传递参数,举例: 网站搜索,跳转等
    1. 黑客构造出特殊url(邮件发给用户个特殊链接),其中包含恶意代码
    2. 用户访问url,黑客指定的服务器返回给用户恶意代码
    3. 用户收到请求数据,立刻开始解析,混在其中的恶意代码也被解析.
    4. 恶意代码可以窃取用户数据发送到指定网站或者冒充用户发送请求
```
解决方案:
防止攻击者提交恶意代码,防止浏览器执行恶意代码
1. 对输出代码进行严格的代码编码,HTML/CSS/JS/URL都进行编码
  vue-react 都不使用类似于 v-html等语句
2. 路径访问设定白名单
3. 一些常见输入设定输入类型,简单效验
4. 禁止JavaScript读取cookie(就算得到cookie也无法读取)


补充: iframe攻击
点击劫持: iframe有时候会被一个类似于遮罩层(透明度为0)的页面遮罩,用户的点击都会落在其他页面上
解决办法: 禁止iframe上外链其他js
